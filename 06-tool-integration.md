# Tool Integration Guide - 工具链集成指南

> **模块定位**: 设计工具与代码之间的桥梁，Design Token 工作流
> **依赖关系**: [05-platform-decision-matrix.md](./05-platform-decision-matrix.md)（平台 Token 映射）
> **预估行数**: ~180 行

---

## 🎯 核心理念

**"Single Source of Truth"** — Design Token 是唯一真实来源。

从设计工具到代码的完整链路：

```
Figma/Sketch 设计
    ↓ 导出
Design Token (JSON/W3C 格式)
    ↓ 转换
Platform-specific 代码 (Swift/Kotlin/CSS/React)
    ↓ 集成
App / Web 产品
```

## 2. Design Token Export Formats（Token 导出格式）

### W3C Community Group 标准（推荐）

```json
// tokens.json (W3C Design Token Format Module Level 1)
{
  "$schema": "https://design-tokens.github.io/community-group/format/",
  "color": {
    "brand": {
      "primary": {
        "$type": "color",
        "$value": "#FFD000"
      },
      "primary-light": {
        "$type": "color",
        "$value": "#FFF8E0"
      }
    }
  },
  "dimension": {
    "spacing": {
      "md": {
        "$type": "dimension",
        "$value": "16px"
      }
    }
  },
  "typography": {
    "font": {
      "body": {
        "$type": "typography",
        "$value": {
          "fontFamily": "SF Pro Text",
          "fontSize": "16px",
          "fontWeight": 400,
          "lineHeight": "1.5"
        }
      }
    }
  }
}
```

### Style Dictionary 格式（工程化首选）

```json
// tokens/properties.json
{
  "color": {
    "brand": {
      "primary": { "value": "#FFD000" },
      "primary-light": { "value": "#FFF8E0" },
      "secondary": { "value": "#FF7E00" }
    },
    "bg": {
      "page": { "value": "#F5F5F5" },
      "card": { "value": "#FFFFFF" }
    }
  },
  "spacing": {
    "xs": { "value": "8px" },
    "sm": { "value": "12px" },
    "md": { "value": "16px" },
    "lg": { "value": "24px" }
  }
}
```

```javascript
// style.dictionary.config.js
module.exports = {
  source: ["tokens/**/*.json"],
  platforms: {
    // iOS Swift 输出
    ios: {
      transformGroup: "ios-swift",
      buildPath: "build/ios/",
      files: [{
        destination: "DesignToken.swift",
        format: "ios-swift/separated-enum",
        className: "DesignToken"
      }]
    },
    // Android Kotlin 输出
    android: {
      transformGroup: "android",
      buildPath: "build/android/",
      files: [{
        destination: "res/values/design_tokens.xml",
        format: "android/resources"
      }]
    },
    // Web CSS 输出
    web: {
      transformGroup: "web",
      buildPath: "build/web/",
      files: [{
        destination: "variables.css",
        format: "css/variables"
      }, {
        destination: "tokens.module.js",
        format: "javascript/es6"
      }]
    }
  }
};
```

## 3. Platform-Specific Code Generation（平台特定代码生成）

### iOS (Swift) 输出示例

```swift
// Generated by Style Dictionary
// File: DesignToken.swift

import SwiftUI

public enum DesignToken {
    public enum Color {
        public static let brandPrimary = Color(hex: "FFD000")
        public static let brandPrimaryLight = Color(hex: "FFF8E0")
        public static let brandSecondary = Color(hex: "FF7E00")

        public enum Bg {
            public static let page = Color(hex: "F5F5F5")
            public static let card = Color(hex: "FFFFFF")
        }

        public enum Text {
            public static let primary = Color(hex: "1A1A1A")
            public static let secondary = Color(hex: "666666")
        }

        public enum Feedback {
            public static let error = Color(hex: "F44336")
            public static let success = Color(hex: "4CAF50")
        }
    }

    public enum Spacing {
        public static let xxs: CGFloat = 4
        public static let xs: CGFloat = 8
        public static let sm: CGFloat = 12
        public static let md: CGFloat = 16
        public static let lg: CGFloat = 24
        public static let xl: CGFloat = 32
        public static let xxl: CGFloat = 48
    }

    public enum Typography {
        public static let hero: Font = .system(size: 32, weight: .bold)
        public static let title: Font = .system(size: 24, weight: .semibold)
        public static let body: Font = .system(size: 16, weight: .regular)
        public static let caption: Font = .system(size: 13, weight: .regular)
    }
}
```

## 4. Icon Sync Workflow（图标同步工作流）

### 从 Figma 到代码

```
Step 1: 在 Figma 中使用 Icon 组件库
    ↓ （推荐使用 Iconify 组件集）
Step 2: 标注图标名称和库名
    ↓ （如：icon-park-solid:home）
Step 3: 导出时自动匹配图标 ID
    ↓
Step 4: 生成多框架代码
    ↓ （参考 [03-icon-system.md] Layer 4）
Step 5: 集成到项目中
```

### 推荐的 Figma 图标组件库

| 库名 | Figma 社区链接 | 说明 |
|------|----------------|------|
| **Iconify** | 搜索 "Iconify Icons" | 包含所有主流图标库 |
| **Phosphor Icons** | 官方提供 | Phosphor 全系列 |
| **Lucide Icons** | 社区维护 | Lucide 最新版 |
| **Material Symbols** | Google 官方 | Material 图标 |
| **IconPark** | 字节官方 | 中文场景优化 |

### 自动化脚本示例

```bash
#!/bin/bash
# sync-icons.sh - 图标同步脚本

echo "🔄 开始同步图标..."

# 1. 从 Figma API 导出图标组件列表
# （需要 Figma Access Token）
figma-export icons \
  --file-id=YOUR_FILE_ID \
  --output=icons/metadata.json

# 2. 生成各框架代码
npx iconify-codegen \
  --input=icons/metadata.json \
  --output=src/icons/ \
  --frameworks=react,vue,swift,kotlin

# 3. 验证图标完整性
node scripts/verify-icons.js

echo "✅ 图标同步完成！"
```

## 5. Version Control for Design Assets（设计资源版本控制）

### 推荐目录结构

```
project-root/
├── design-tokens/              # Design Token 源文件
│   ├── global/                 # 全局 Token
│   │   ├── colors.json
│   │   ├── typography.json
│   │   ├── spacing.json
│   │   └── radius.json
│   ├── platforms/              # 平台特定覆盖
│   │   ├── ios/
│   │   ├── android/
│   │   └── web/
│   └── themes/                 # 主题变体
│       ├── light.json
│       └── dark.json
│
├── generated/                  # 生成的代码（不提交到 Git）
│   ├── ios/
│   ├── android/
│   └── web/
│
└── assets/                     # 静态资源
    ├── icons/                  # SVG 图标
    ├── images/                 # 图片
    └── fonts/                  # 字体文件
```

### Git 工作流

```bash
# 仅提交源文件，不提交生成的代码
echo "design-tokens/" >> .gitignore
echo "generated/" >> .gitignore

# 但保留 README 和配置文件
!design-tokens/*.md
!design-tokens/*.config.js

# CI/CD 时自动生成
# .github/workflows/generate-tokens.yml
```

## 6. Collaboration Tools Integration（协作工具集成）

### Design → Development Handoff（设计到开发交付）

| 工具 | 适用场景 | 与本 Skill 的结合点 |
|------|----------|---------------------|
| **Zeplin** | 标注、切图、Token 导出 | 直接导出 Style Dictionary 格式 |
| **Figma Dev Mode** | 实时代码预览、测量 | 复制 Token 名称而非硬编码值 |
| **Storybook** | 组件文档、测试 | 使用 Token 构建组件变体 |
| **Zeroheight** | 设计规范文档 | 自动生成本 Skill 的文档版本 |

### Code Review Checklist for Design Implementation

开发者在实现设计时，检查：

- [ ] 是否使用 `DesignToken.Color.xxx` 而非硬编码颜色？
- [ ] 是否使用 `DesignToken.Spacing.xxx` 而非魔法数字？
- [ ] 字体大小是否来自 `DesignToken.Typography`？
- [ ] 圆角值是否使用 Token？
- [ ] 暗色模式是否使用对应的 Token 变体？
- [ ] 图标是否遵循 [03-icon-system.md](./03-icon-system.md) 的命名规范？
