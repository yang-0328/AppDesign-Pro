## 标准屏幕模板

### Onboarding（引导页）

**适用场景**: 首次启动、功能介绍、价值主张展示

```
┌─────────────────────────────────────┐
│                                     │
│         [跳过]              ● ○ ○   │  ← 进度指示器 + 跳过
│                                     │
│                                     │
│          🎨 [插图/动画]             │  ← 视觉焦点
│                                     │
│     "发现身边的美食"                │  ← 价值主张标题
│   "3秒下单，30分钟送达"            │  ← 简短描述
│                                     │
│         [开始使用]                  │  ← 单一 CTA
│                                     │
└─────────────────────────────────────┘
```

**规范**:
- 3–5 屏，可跳过，带进度指示器
- 聚焦**价值主张**，非功能列表
- 每屏**单一 CTA**
- 权限请求在**上下文中**提出，不前置

### Authentication（认证）

**适用场景**: 登录、注册、找回密码

```
┌─────────────────────────────────────┐
│                                     │
│        [Logo / 品牌名]              │
│                                     │
│    ┌─────────────────────────┐      │
│    │ 📧 手机号/邮箱           │      │  ← 最小字段集
│    └─────────────────────────┘      │
│    ┌─────────────────────────┐      │
│    │ 🔒 密码                   │      │
│    └─────────────────────────┘      │
│                                     │
│    [       登录       ]            │  ← 主按钮
│                                     │
│  ──── 或用以下方式登录 ────         │  ← 社交登录
│  [微信]  [Apple]  [Google]          │
│                                     │
│  忘记密码？  新用户注册               │  ← 次要操作
│                                     │
└─────────────────────────────────────┘
```

**规范**:
- 社交登录 + 邮箱/手机
- **尊重平台默认**：iOS 优先 Sign in with Apple
- 首次登录后启用**生物识别**
- 最小字段：邮箱/手机 + 密码
- 注册后**渐进式完善资料**

### Feed / List（信息流/列表）

**适用场景**: 首页Feed、商品列表、消息列表

```
┌─────────────────────────────────────┐
│ [筛选▼] [排序▼]        [视图切换]   │  ← 粘性筛选栏
├─────────────────────────────────────┤
│ ┌──────┐ ┌──────┐ ┌──────┐        │
│ │▓▓▓▓▓▓│ │▓▓▓▓▓▓│ │▓▓▓▓▓▓│        │  ← 骨架屏 (Loading)
│ └──────┘ └──────┘ └──────┘        │
│                                     │
│ ┌──────────────────────────────┐   │
│ │ [图片]  标题                 │   │  ← 卡片项
│ │         描述文字...          │   │
│ │         标签1  标签2  ¥99    │   │
│ └──────────────────────────────┘   │
│ ┌──────────────────────────────┐   │
│ │ [图片]  标题                 │   │
│ │         ...                  │   │
│ └──────────────────────────────┘   │
│                                     │
│         [加载更多...]              │  ← 底部加载
└─────────────────────────────────────┘
```

**规范**:
- 无限滚动 + **虚拟列表**（性能优化）
- 初始加载显示**骨架屏占位符**
- 下拉刷新 + 底部加载更多
- 适用时显示**粘性筛选/排序栏**

### Detail View（详情页）

**适用场景**: 商品详情、文章详情、用户主页

```
┌─────────────────────────────────────┐
│ [←返回]  [分享]  [收藏]            │  ← 导航栏
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────────┐   │
│  │                             │   │
│  │      Hero 图片/视频          │   │  ← 视差滚动
│  │     (16:9 或全宽)           │   │
│  │                             │   │
│  └─────────────────────────────┘   │
│                                     │
│  商品名称                           │  ← 标题区
│  ★★★★☆ 4.8 | 月售 2345           │
│  ¥128                              │
│                                     │
│  ─── 详细信息 ───                  │
│  规格、参数、描述...                │
│                                     │
│  ─── 相关推荐 ───                  │
│  [卡片] [卡片] [卡片] 横向滚动      │
│                                     │
├─────────────────────────────────────┤
│  [加入购物车]    [立即购买]        │  ← 底部操作栏(固定)
└─────────────────────────────────────┘
```

**规范**:
- 顶部 Hero 图/视频带**视差滚动**
- 底部**固定操作栏**放置主要 CTA
- 包含"相关推荐"/"猜你喜欢"区块

### Profile / Settings（个人中心/设置）

**适用场景**: 用户设置、应用偏好、账户管理

```
┌─────────────────────────────────────┐
│  ← 设置                             │
├─────────────────────────────────────┤
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 👤 头像   用户名      >     │   │  ← 账户区域
│  └─────────────────────────────┘   │
│                                     │
│  账户设置                           │  ← 分组标题
│  ├─ 个人资料                 >     │
│  ├─ 账户安全                 >     │
│  └─ 绑定第三方               >     │
│                                     │
│  偏好设置                           │
│  ├─ 通知设置                 >     │
│  ├─ 隐私设置                 >     │
│  └─ 通用设置                 >     │
│                                     │
│  其他                               │
│  ├─ 关于我们                 >     │
│  ├─ 用户协议                 >     │
│  └─ [退出登录]  (红色/警告色)      │  ← 破坏性操作
│                                     │
└─────────────────────────────────────┘
```

**规范**:
- 分组设置，清晰的分组标题
- **账户管理**置顶，**偏好**居中，**法律/关于**置底
- **破坏性操作**明确分离并需二次确认

### Search（搜索）

**适用场景**: 全局搜索、站内搜索、筛选查找

```
┌─────────────────────────────────────┐
│ [←取消] ┌──────────────────┐[🎤]  │
│          │ 🔍 搜你想吃的    │      │  ← 搜索框 + 语音
│          └──────────────────┘      │
├─────────────────────────────────────┤
│  🔥 热门搜索                        │  ← 聚焦时显示
│  [美团外卖] [肯德基] [星巴克]       │
│                                     │
│  🕐 历史记录                        │
│  [火锅] [披萨] [日料]    [清空]     │
│                                     │
│  ─── 搜索结果 ───                  │  ← 输入后实时过滤
│  美团外卖 (12个结果)                │
│  美团外卖优惠券 (3个结果)           │
│  美团外卖骑手 (1个结果)             │
│                                     │
│  未找到相关结果                     │  ← 空状态
│  尝试其他关键词或清除筛选条件       │
│  [清除所有筛选]                     │
└─────────────────────────────────────┘
```

**规范**:
- 聚焦时显示**历史记录 + 热门搜索**
- 输入时**实时过滤**建议
- 提供**语音输入**选项
- 空结果显示**建议**，而非死胡同

## Output Format（输出格式标准）

### Step 1: Understand（理解需求）

- 明确目标平台（iOS / Android / Cross-platform）
- 识别核心用户流程和**主要操作**
- 需求模糊时提出**一个**澄清问题

### Step 2: Design Specification（设计规格说明）

```markdown
## [Screen/Feature Name]

### User Story
As a [user], I want to [action], so that [outcome].

### Information Architecture
- [层级和内容结构]

### Visual Spec
- Layout: [网格/间距描述]
- Typography: [type scale 使用]
- Color: [语义 Token 使用]
- Key components: [组件列表]

### Interactions
- [手势/动画/过渡描述]
- [状态转换]

### Edge Cases
- Empty state / Error state / Loading state / Offline state
```

### Step 3: Implementation（实现代码）

- **完整、生产级**代码
- 使用要求的框架/语言
- 包含**所有状态**（loading, empty, error, success）
- 在注释中标注关键设计决策
- 包含无障碍标签和提示
- 按 Layer 6 标准添加图标注释

### Step 4: Quality Checklist（质量检查清单）

最终确认前验证：

- [ ] 所有触摸目标 ≥ 44pt / 48dp
- [ ] 使用语义颜色 Token（组件中无硬编码 hex）
- [ ] 支持暗色模式
- [ ] 设计了 Loading、Empty、Error 状态
- [ ] 动态字体/字体缩放正常工作
- [ ] 无障碍标签完整
- [ ] 动画尊重系统设置
- [ ] 安全区域/刘海处理正确
- [ ] 图标库一致（最多混用 2 个）
- [ ] 图标容器符合设计系统
- [ ] 所有图标都标注了 fallback

## Performance Budget（性能预算）

| 指标 | 目标值 | 说明 |
|------|--------|------|
| First Meaningful Paint | < 1.5s | 首次有意义的绘制 |
| Interaction Response | < 100ms | 交互响应时间 |
| Animation Frame Budget | 16.6ms | 动画帧预算 (60fps) |
| Bundle Size | < 200KB (gzipped) | JS Bundle 大小 |
| Image Optimization | WebP/AVIF | 图片格式优化 |
| Cache Strategy | Stale-While-Revalidate | 缓存策略 |

## Accessibility (a11y) Quick Reference（无障碍速查）

- 每个交互元素：无障碍标签、角色、状态
- 动态字体支持：UI 在 200% 缩放时不崩溃
- **颜色不是唯一**的信息传达方式
- WCAG AA 对比度最低要求
- 屏幕阅读器导航顺序与视觉顺序一致
- 焦点管理：模态框中捕获焦点，关闭时返回焦点
